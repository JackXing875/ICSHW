# Homework6 参考答案

分配器：隐式空闲链表 + 边界标记（header/footer 各 4B，共 8B 开销），
double word（8B）对齐。图中每格 4B，标注 X/Y 表示块总大小 X 字节、
分配位 Y（1 已分配、0 空闲）。

初始堆从左到右共 5 个块（总大小含 header/footer）：

```text
[ 32/0 free ][ 16/1 p1 ][ 16/0 free ][ 16/1 p2 ][ 24/0 free ]
```

p1、p2 块大小 16（payload 8 + 开销 8），与题意一致。堆总大小
= 32 + 16 + 16 + 16 + 24 = 104 字节。

四条语句各自需要的块大小（payload + 8，向上取 8 的倍数）：

```text
malloc(4):  4 + 8  = 12 -> 16
malloc(10): 10 + 8 = 18 -> 24
malloc(16): 16 + 8 = 24 -> 24
```

## Problem 1

初始堆状态即上图：一个 32 的空闲块、p1 指向的 16 的已分配块、一个 16 的空闲块、
p2 指向的 16 的已分配块、一个 24 的空闲块；并据上面算出 4 次 malloc 分别要 16、
24、24 的块，free(P1) 释放 p1 那个 16 的块。下一题据此画出两种策略的演化。

## Problem 2

逐句画出 first fit 与 best fit 下的内存分布（F=free，A=allocated）。

### First fit

```text
初始:  [F32][A16 p1][F16][A16 p2][F24]

1 P3=malloc(4)  需 16：第一个空闲块 F32 命中，切成 A16(P3)+F16
       [A16 P3][F16][A16 p1][F16][A16 p2][F24]

2 P4=malloc(10) 需 24：前两个 F16 都不够，F24 命中(正好)
       [A16 P3][F16][A16 p1][F16][A16 p2][A24 P4]

3 free(P1)      释放 p1 那个 A16，与左右两个 F16 合并成 F48
       [A16 P3][F48][A16 p2][A24 P4]

4 P5=malloc(16) 需 24：F48 命中，切成 A24(P5)+F24
       [A16 P3][A24 P5][F24][A16 p2][A24 P4]
```

### Best fit

```text
初始:  [F32][A16 p1][F16][A16 p2][F24]

1 P3=malloc(4)  需 16：F32/F16/F24 中最合适的是 F16(正好)
       [F32][A16 p1][A16 P3][A16 p2][F24]

2 P4=malloc(10) 需 24：F32/F24 中最合适的是 F24(正好)
       [F32][A16 p1][A16 P3][A16 p2][A24 P4]

3 free(P1)      释放 p1 那个 A16，只与左边 F32 合并成 F48（右边已分配）
       [F48][A16 P3][A16 p2][A24 P4]

4 P5=malloc(16) 需 24：F48 命中，切成 A24(P5)+F24
       [A24 P5][F24][A16 P3][A16 p2][A24 P4]
```

### 峰值利用率

利用率 = 当前已分配 payload 之和 / 堆大小。整个过程中堆始终是 104 字节
（两种策略都能在已有空闲块中放下，无需扩展堆）。逐句的 payload 之和：

```text
初始        p1+p2            = 8+8            = 16
1 malloc(4)  +P3(4)          = 8+8+4         = 20
2 malloc(10) +P4(10)         = 8+8+4+10      = 30
3 free(P1)   -p1(8)          = 8+4+10        = 22
4 malloc(16) +P5(16)         = 8+4+10+16     = 38
```

峰值 payload = 38（出现在第 4 句后）。

峰值利用率 = 38 / 104 ≈ 0.365 = 36.5%（first fit 与 best fit 相同，因为两者堆大小
都是 104、各时刻 payload 相同）。
